# Telegram backup bot pour Railway

Ce bot fait exactement ce que tu as demandé :

- surveille un **groupe source**
- **copie automatiquement** chaque vidéo vers un **canal privé de backup**
- enregistre les métadonnées dans **Postgres**
- peut **republier** plus tard tout ou partie du backup dans un **nouveau groupe** sur commande admin

## Fonctionnement

Flux conseillé :

1. **Groupe source** : les membres envoient des vidéos.
2. **Canal privé backup** : le bot copie automatiquement chaque vidéo.
3. **Nouveau groupe** : si l’ancien saute, tu ajoutes le bot au nouveau groupe et tu relances la publication.

## Commandes

- `/status` → voir l’état du bot
- `/bindtarget` → à lancer **dans le nouveau groupe cible**
- `/publish_last 50` → republie les 50 derniers médias sauvegardés
- `/publish_since 2026-04-01` → republie depuis une date
- `/publish_all` → republie tout

## Pré-requis Telegram

1. Crée le bot avec **@BotFather**.
2. Ajoute le bot comme **admin** dans :
   - le groupe source
   - le canal privé backup
   - le nouveau groupe cible
3. Désactive le **privacy mode** si nécessaire dans BotFather pour que le bot voie correctement les messages du groupe.
4. Récupère les IDs de chat :
   - `SOURCE_CHAT_ID`
   - `BACKUP_CHAT_ID`

## Variables d’environnement

Copie `.env.example` puis remplis :

- `BOT_TOKEN`
- `DATABASE_URL`
- `SOURCE_CHAT_ID`
- `BACKUP_CHAT_ID`
- `ADMIN_USER_IDS`
- `WEBHOOK_PATH_SECRET`
- `TELEGRAM_SECRET_TOKEN`
- `WEBHOOK_URL` **ou** laisse Railway fournir `RAILWAY_PUBLIC_DOMAIN`

## Déploiement Railway

1. Crée un projet Railway.
2. Ajoute un service **PostgreSQL**.
3. Déploie ce repo comme service web Python.
4. Active le **public networking** pour obtenir un domaine public.
5. Ajoute les variables d’environnement.
6. Vérifie que le service démarre bien.
7. Le bot va essayer d’enregistrer son webhook automatiquement au démarrage.

## Utilisation

### 1. Backup auto

Dès qu’une vidéo arrive dans le groupe source, le bot la copie dans le canal backup.

### 2. Préparer un nouveau groupe

- crée le nouveau groupe
- ajoute le bot en admin
- dans ce nouveau groupe, lance :

```text
/bindtarget
```

### 3. Republier

Depuis ton chat privé avec le bot ou depuis un groupe où tu es admin :

```text
/publish_last 50
```

ou

```text
/publish_all
```

## Notes techniques

- le bot gère `video`
- il gère aussi les fichiers `document` dont le MIME type commence par `video/`
- il stocke aussi `media_group_id` pour garder la trace des albums
- la republication se fait depuis le **canal backup**, pas depuis le groupe source

## Limites / améliorations possibles

Version actuelle :

- sauvegarde les vidéos et documents vidéo
- republie en séquence simple

Améliorations faciles :

- `/publish_range 2026-04-01 2026-04-05`
- `/pause_backup` et `/resume_backup`
- anti-doublons plus strict sur les albums
- panneau admin plus propre
- filtre par user
- republish album par album

## Lancement local

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Ensuite expose ton serveur local avec un tunnel HTTPS et définis `WEBHOOK_URL`.
